Generating Standalone Pages

Note : This functionality is only relevant to the local filesystem version of Trident Sandbox, for Hosted or AppCached environments you would use a similar 'Launch Command using SandboxLoader' functionality (See Hosting and AppCache page).

As of version 1.5, Trident Sandbox includes a feature to generate a standalone web page.  You can use this to pick a template to use for generating a standalone web page which is just your program, automatically loaded.  You will still need to enable scripts but you will not be cluttered with the ide... the Output Areas are full screen.  You will also not have to jump through the hoops of clicking 'Load', browsing for your PRG file, and then clicking Run.  If you need to make changes, however, you will need to do that from within the TridentSandbox main page and then re-generate a standalone html. 

Templates are minimal htm files with parameters in position for substituting your app's html and script.  The parameters used are @TRIDENT_HTML@ and @TRIDENT_SCRIPT@.  All the unneccessary html visual elements and scripts used to support the IDE are removed to allow the Output Area to use the full page.  The templates still include scripts and styles for all the libraries/components made available in the ide.  I do provide an alternate template that includes everything but the editors, in case you are not creating and using the syntax highlighting code editors in your own PRG files... this reduces the number of 'includes' significantly making for a smaller, lighter standalone page. 

Note : Your application will still depend on the folder structure beneath the root folder to get access to the libraries, images, sounds, etc which are needed to make your page work correctly, but it is 'standalone' in the sense that the TridentSandbox.htm page is 'standalone'.  Advanced users can determine ways to minimize dependencies if needed.

The process for generating a standalone web page is :

  • Load your PRG file from within Trident Sandbox, make sure the Program Name displayed is what you want to use for a file name (we automatically add the .htm extension)
  • Click on the 'Generate S/A' button.  This will reveal a file picker allowing you to choose a template to use for page generation.
  • Choose the template you want to use from those available in the templates folder.
  • Upon selecting the template a download process will begin, place the constructed htm file into the same parent directory that TridentSandbox.htm is located.

Now you can run your page, enable scripts and bookmark it for quick access.
