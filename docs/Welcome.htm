Welcome to Trident Sandbox

Trident Sandbox was developed to turn Internet Explorer into a scripting IDE. It is named after the Trident rendering engine within Internet Explorer browsers. Internet Explorer 11 uses Trident engine version 7 (IE 10 uses Trident 6). Trident 6 added the HTML 5 File API which allows loading and saving files to local filesystem. Trident 7 added HTML 5 Fullscreen API and WebGL (among other changes). These capabilities along with a number of open source third-party javascript libraries allow the interactive creation of HTML 5 based applications. Although it was inspired by similar online web tools like jsfiddle, Trident Sandbox is intended to be primarily run from the local filesystem.

There is a pattern of web development called Single Page Application (SPA) in which you attempt to create a web page that changes dynamically vs going from page to page to page. This pattern most closely resembles what goes on in a Trident Sandbox app... you are still on the same page when you run different programs, you are just dynamically building up and tearing down DOM elements from the hierarchical Document Object Model. Your applications are not sandboxed because of the Trident Sandbox main page, they are sandboxed because of Internet Explorer. You can't accidentally change the main page's contents on disk but you can change how it looks/act in the current load of that page. Because of this you may and probably will accidentally mess up the main page from time to time, in which case a reload of the main page will restore to a fresh slate.

The User Area where the (currently) three tabs : Main Output, Log (HTML), and Log (Text) are located is an attempt to create a flexible enough environment to handle simple programs as well as more complicated ones. In theory the main output would be used for mostly static HTML closely resembling what is defined in your Markup area. The Log (HTML) can be used either as a log screen (supporting HTML styling) or you can use it for your own dynamically rendered web pages. You can dynamically create textboxes, list boxes, buttons, canvas objects, media object, dump a large chart there, etc. The Log (Text) tab is where you might log pure text messages... it contains a scrollable textarea and has easy API interface.

To help simplify your ability to manipulate the sandbox, i have provided a set of API functions, user elements, and variables which can help you control the environment.

A simple object inspector is provided which will allow you to examine variables in your program. If you need more debugging capabilities, you can use the Internet Explorer script debugger to debug your scripts. Since your code is dynamic you need to place a javascript command 'debugger;' (no quotes) into a line of code in your script and then pressing F12 to open the Internet Explorer tools and find the tab with a bug icon picture on it to enable the script debugger... leaving the script debugger window open attempt your run your program and when it hits that line of code you can inspect variables, set other breakpoints, or access the javascript console.

Trident Sandbox comes with a number of open source third-party javascript libraries which you can use within your programs. These libraries (as of Trident Sandbox version 1.75) include :

  • jQuery : like a query language for the html DOM. It is also needed by most of the other libraries.
  • jQuery UI : provides some common controls like datepickers, dialog boxes, etc and supports themes.
  • jqGrid : a popular versatile grid control
  • jqPlot : awesome charting library
  • Alertify.JS : Simple way of dispaying message boxes or notifications.
  • Crypto.JS : Simple encryption capabilities
  • Dynatree : a hierarchical tree control
  • Three.JS : Bare minimum distribution included, download the full zip file from their website. Allows higher level use of WebGL.
  • Loki.JS : A lightweight javascript document oriented database
  • Metro.JS : Lets you create Metro-like live tiles within your programs.
  • Font Awesome : Font based vector icon library... these will scale well.
  • EaselJS : A Javascript library that makes working with the HTML5 Canvas element easy
  • Pixi.js : 2D webGL renderer with canvas fallback
  • Springy : A force directed graph layout algorithm in JavaScript
  • Math.js : An extensive math library for JavaScript.  Inspect mathjs() and look at docs property for interactive help.
  • Buckets.js : A complete, fully tested and documented data structure library written in JavaScript
  • Obelisk.js : Obelisk.js is a JavaScript Engine for building isometric pixel objects
  • TinyMCE : Javascript HTML WYSIWYG editor control
  • Indexed.js : Mongo-like interface for IndexedDB.  ...for if the Trident DB and/or loki.js does not fit your program's database needs

If you need to add other javascript libraries you can put them alongside the others... in the libraries subfolder. Then edit the TridentSandbox.htm file and add script include tags (or stylesheet links) similar to the other libraries.
