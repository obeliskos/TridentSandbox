Welcome to Trident Sandbox

Trident Sandbox was developed to turn your browser into a scripting IDE.  It's current focus is to provide a lightweight development and runtime ecosystem which you can use to run on and create apps for almost any modern device and operating system.  This ecosystem consists of modern browser standards, a sandbox api to provide environment capabilities, along with a curated set of third party libraries.  Originally, this project was created to give new life to Surface RT devices, and while it also offers the ability to use the WinJS library, it remains cross platform.

WinJS

TridentSandbox now includes the WinJS 4.0 framework which provides several touch-friendly UI controls.  To further assist app development, it provides a Promise framework, a javascript job scheduler, data binding, etc.  This library was originally designed to assist in creating Microsoft AppStore apps written in javascript, but this library has been open sourced and positioned to run in cross-browser (non appstore) environments.  There may still be many areas where the UI controls still 'work best' in Internet Explorer, but they are still in active development on GitHub and are continuing to fix bugs.  It adds value to Trident Sandbox by enabling you to create apps that are more touch-friendly and act more like 'apps'.  I now have a 'WinJS' and the older 'Standard' versions of TridentSandbox. Both actually include the WinJS library, but the WinJS version also includes the stylesheet which is needed for the UI controls.  WinJS version is now preferred and depending on cross browser support testing it might even become the default.  If you're inclined to read to learn more, you might read the 'WinJS Controls' section of the free 30meg microsoft ebook here

SPA / Native Web Apps

There is a pattern of web development called Single Page Application (SPA) in which you attempt to create a web page that changes dynamically vs going from page to page to page. This pattern most closely resembles what goes on in a Trident Sandbox app... you are still on the same page when you run different programs, you are just dynamically building up and tearing down DOM elements from the hierarchical Document Object Model.  Your applications are not sandboxed by the Trident Sandbox main page, they are sandboxed by your browser. A more modern description of this type of application might also be Native Web App.

API

To help simplify your ability to manipulate the sandbox, i have provided a class hierarchy starting with the 'sandbox' object. This object namespace contains sandbox specific utility methods for manipulating sandbox resources. More information about this can be found on the API Reference help page, or by clicking the Inspect (eye) icon on an IDE page when no text is selected in an editor.

Libraries

Trident Sandbox comes with a number of open source third-party javascript libraries which you can use within your programs. These libraries (as of Trident Sandbox version 1.87) include :

  • jQuery : like a query language for the html DOM. It is also needed by most of the other libraries.
  • jQuery UI : provides some common controls like datepickers, dialog boxes, etc and supports themes.
  • CodeMirror : syntax-highlighting editors used for markup and script editing within the ide.
  • WinJS : Microsoft javascript library for creating browser 'apps'.
  • React.JS : A (Facebook created) JavaScript library for building user interfaces
  • Knockout.js : Simplify Javascript UIs with with the Model-View-View Model (MVVM) pattern.
  • jqGrid : a popular versatile grid control
  • jqPlot : awesome charting library
  • Alertify.JS : Simple way of dispaying message boxes or notifications.
  • PrettyPrint.js : This is what the Trident 'Inspect' function uses to dump an object to an html table
  • Crypto.JS : Simple encryption capabilities
  • Dynatree : a hierarchical tree control
  • Three.JS : Bare minimum distribution included, download the full zip file from their website. Allows higher level use of WebGL.
  • Loki.JS : A lightweight javascript document oriented database (which I have also contributed to).
  • Metro.JS : Lets you create Metro-like live tiles within your programs.
  • Font Awesome : Font based vector icon library... these will scale well.
  • EaselJS : A Javascript library that makes working with the HTML5 Canvas element easy
  • Pixi.js : 2D webGL renderer with canvas fallback
  • Springy : A force directed graph layout algorithm in JavaScript
  • Obelisk.js : Obelisk.js is a JavaScript Engine for building isometric pixel objects
  • TinyMCE : Javascript HTML WYSIWYG editor control
  • Indexed.js : Mongo-like interface for IndexedDB.  ...for if the Trident DB and/or loki.js does not fit your program's database needs
  • Moment.js : A javascript date library for parsing, validating, manipulating, and formatting dates.
  • Babylon.js : a complete JavaScript framework for building 3D games with HTML 5 and WebGL.
  • jsoneditor.js : JSON Editor takes a JSON Schema and uses it to generate an HTML form.
  • d3.js : Javascript visualization library.

If you need to add other javascript libraries you can put them alongside the others... in the libraries subfolder. Then edit the TridentSandbox.htm file and add script include tags (or stylesheet links) similar to the other libraries.
